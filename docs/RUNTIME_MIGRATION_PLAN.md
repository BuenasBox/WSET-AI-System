## Verified Architecture

### Two-Repository Model

| Repository | Role | Deployed at |
|------------|------|-------------|
| `WSET-AI-System-push` | Cognitive backend, data generators, tests | local only |
| `epistemiclab-dashboard` | Static GitHub Pages site | `epistemiclab.dpdns.org` |

## Route-by-Route Architecture

### `/diagnostic-sba/`
- **Type:** Static. Loads `preguntas.json` from same directory.
- **Data source:** `preguntas.json` generated by `tools/question_generation/static_demo_exporter.py` from the diagnostic SBA draft/review pipeline.
- **Current inventory:** 36 items (all `approved_for_static_demo` reviews exhausted).
- **Pipeline state:** EXHAUSTED. 16 items are `preserve_only`, 2 are `requires_revision`. No further items can be exported without new enrichment drafts and human review records.
- **Migration verdict:** NO EXPANSION POSSIBLE through existing pipeline. Diagnostic SBA stays at 36 items.
- **Changes required:** WSET-AI-System = none. epistemiclab-dashboard = none.

### `/adaptive-session/`
- **Type:** Static. Fetches `../session_data/session_payload.json` at runtime.
- **Data source:** `session_payload.json` generated by `tools/question_generation/generate_session_payload.py` → calls `load_master_bank()` + `select_sba_session_items(collection='public_lab')`.
- **Current inventory:** 36 items (master_bank `public_lab` collection only).
- **Available for expansion:** 83 `approved_private_sba` items in master_bank — reviewed and approved, currently in `inactive` collection only.
- **Migration:** Promote 83 `approved_private_sba` items to `public_lab` collection → regenerate `session_payload.json` with questions drawn from expanded 119-item pool.
- **Changes required:**
  - WSET-AI-System: `knowledge/question-bank/master_bank/master_bank.json` (update collections); `frontend/session_data/session_payload.json` (regenerate).
  - epistemiclab-dashboard: `session_data/session_payload.json` (copy new file).
- **Deployment order:** WSET-AI-System first, then copy to dashboard.

### `/open-response-lab/`
- **Type:** Static. Loads `lab_payload.js` as an inline script (`window.OPEN_RESPONSE_LAB_PAYLOAD`).
- **Current state:** `activation_status: inactive`, 10 items hardcoded.
- **Available for expansion:** `knowledge/question-bank/open_response/open_response_bank.json` has 20 items; all 26 OR review records have `review_status: approved` and `activation_status: inactive`.
- **Migration:** Regenerate `lab_payload.js` from full 20-item bank. Set `activation_status: active_private_lab`. Embed Phase X.1 assessment intelligence as `assessment_intelligence` top-level key.
- **Changes required:**
  - WSET-AI-System: generate new `frontend/open-response-lab/lab_payload.js`.
  - epistemiclab-dashboard: `open-response-lab/lab_payload.js` (copy new file).
- **Deployment order:** WSET-AI-System first, then copy to dashboard.

### `/` (root dashboard)
- **Type:** Static. Fetches `./system_state.json`.
- **Current state:** `system_state.json` contains stale counts and missing Phase X.1/X.2 status fields.
- **Migration:** Update `system_state.json` with accurate counts and new status fields.
- **Changes required:**
  - WSET-AI-System: `frontend/architecture-dashboard/system_state.json`.
  - epistemiclab-dashboard: `system_state.json` (copy new file).
